# Правила для системы контроля версий

Команда использует Git. Основная ветка main содержит только production ready код, который прошел все тесты. Ветка develop содержит основной код продукта и в промежутке между слияниями в main обозначает разработку новой версии. 

Дополнительные ветки создаются под добавление функционала. Каждая ветка должна содержать один из префиксов:
* feature/
* hotfix/
* test/
* experimental/

Каждый коммит должен содержать номер задачи и краткое сообщение, описывающее изменения. Коммит должен содержать изменения, которые связаны логически одной темой. Формат сообщения: `[TASK-ID] Commit message`.
