# Архитектура приложений

Команда предпочитает следующие разновидности архитектур программного обеспечения:
* layer architecture
* hexagonal architecture
* MVC

По умолчанию следует использовать layer architecture. Для этого выделяются сущности в доменной области целевого приложения, для них создаются модели. Если приложение хранит данные, то модели должны быть созданы после проектирования схемы базы данных или другого формата хранения. После уровня моделей идет уровень сервисов. Сервисы модифицируют модели и выполняют функциональность. Самый высший уровень содержит транспортные фреймворки или интерфейс взаимодействия с пользователями. 

Вариант с hexagonal architecture подходит, если задача приложения масштабная, содержит множество доменов и по несколько вариантов транспорта, интерфейса или хранения данных на разных уровнях. В целом если масштаб приложения велик и оно имеет большое количество сценариев использования, то стоит использовать этот подход.

Model View Controller подходит для несложных GUI приложений, задача которых предоставить интерфейс взаимодействия с данными для пользователя.
